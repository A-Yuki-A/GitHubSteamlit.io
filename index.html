アプリ 公開手順ガイド

GitHub と Streamlit Community Cloud を使って、PythonアプリをWeb公開する手順をまとめたページです。

0. 全体の流れ

  1. GitHubにリポジトリ（プロジェクト用フォルダ）を作成する
  2. リポジトリの中に app.py と requirements.txt を作成する
  3. Streamlit Community Cloud でアプリをデプロイ（公開）する

1. 事前準備

  • GitHubアカウントを作成する（https://github.com/）
  • Streamlitアカウントを作成する（https://streamlit.io/ → GitHubでログイン）
  • GitHub や Streamlit のアカウント登録には、個人のメールアドレスを使用してください。
    例：Gmail、Yahooメールなど。
    学校のメールアドレスや学校アカウント（例：〇〇高校ドメイン、oughアカウント）は使わないでください。
    学校アカウントは外部サービスとの連携に制限があり、 GitHub・Streamlit にログインできない場合があります。

2. GitHubでリポジトリを作る

2-1. リポジトリとは？

リポジトリ（repository）とは、GitHub上に作る「プロジェクト用フォルダ」です。
アプリや作品ごとにリポジトリを分けることで、ファイルを整理して保存できます。

2-2. リポジトリ作成手順

  1. GitHubにログインし、右上の「＋」→ New repository をクリックする
  2. Repository name に、例：my-streamlit-app を入力する
    ※英数字と「-（ハイフン）」を使うのがおすすめです
  3. Public を選択する
    ※Publicは「誰でも見られる設定」です。公開して困る内容は置かないようにしましょう
  4. Add a README file にチェックを入れる
    ※READMEは、このリポジトリの説明を書くメモです（あとから編集できます）
  5. Create repository をクリックして作成する

3. アプリ用ファイルと拡張子の説明

3-1. 拡張子とは？

ファイル名の最後の .py や .txt を拡張子と呼びます。 拡張子は「そのファイルが何の種類か」を表します。

  • .py（Pythonファイル）
    Pythonのプログラムを書くファイルです。 Streamlitアプリ本体は app.py に書きます。
  • .txt（テキストファイル）
    文字だけを保存するファイルです。 Streamlitで使うライブラリの一覧を requirements.txt に書きます。

3-2. app.py のサンプルコード

import streamlit as st

st.title("勇者のステータス表示アプリ")

namae = st.text_input("勇者の名前を入力してください")
level = st.slider("レベル", 1, 99, 10)
hp = st.number_input("HP", min_value=1, max_value=999, value=100)

if st.button("ステータスを表示"):
    st.write("=== 勇者のステータス ===")
    st.write("名前：", namae)
    st.write("レベル：", level)
    st.write("HP：", hp)

3-3. requirements.txt の内容

streamlit

3-4. 生成AIを使ってコードや説明文も作れる

Streamlitアプリのコードや説明文は、生成AI を使うと作成の手助けになります。 代表的なAIサービスは次の3つです：

  • ◆ ChatGPT（OpenAI）
    コード作成、エラー原因の説明、文章作成が得意です。
    公式サイト： https://chat.openai.com
  • ◆ Gemini（Google）
    説明や要約が得意で、調べ学習にも向いています。
    公式サイト： https://gemini.google.com
  • ◆ Claude（Anthropic）
    長い文章やコードの読み取り・添削が得意です。
    公式サイト： https://claude.ai

生成AIは アイデア出し・コードのたたき台作り・エラー調査 に便利です。 ただし、間違った内容が混ざることもあるので、必ず実行して確認しましょう。

4. GitHub上でファイルを作成する（Create new file）

4-1. app.py を作成

  1. 作成したリポジトリのページを開く
  2. 右上の Add file → Create new file を選択する
  3. File name に app.py と入力する
  4. 下のテキストエリアにコードを貼り付ける
  5. 下部の Commit new file を押して保存する

4-2. requirements.txt を作成

  1. Add file → Create new file を選択する
  2. File name に requirements.txt と入力する
  3. 次の1行を入力する：
streamlit
  1. Commit new file を押して保存する

5. Streamlit Community Cloudで公開する

  1. Streamlit Community Cloud にアクセスし、GitHubでログインする
  2. New app をクリックする
  3. Repository に ユーザー名/リポジトリ名（例：testuser/my-streamlit-app）を選択する
  4. Branch は main、Main file path は app.py を指定する
  5. Deploy を押すと公開URL（https://～.streamlit.app）が発行される
