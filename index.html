Streamlitアプリ 公開手順ガイド

GitHub と Streamlit Community Cloud を使って、PythonアプリをWeb公開する手順をまとめたページです。

0. 全体の流れ

  1. GitHubにリポジトリ（プロジェクト用フォルダ）を作成する
  2. リポジトリの中に app.py と requirements.txt を作成する
  3. Streamlit Community Cloud でアプリをデプロイ（公開）する
  4. （任意）この説明ページを GitHub Pages で公開する

1. 事前準備

  • GitHubアカウントを作成する（https://github.com/）
  • Streamlitアカウントを作成する（https://streamlit.io/ → GitHubでログイン）
  • GitHub や Streamlit のアカウント登録には、必ず個人のメールアドレスを使うこと。
    例：Gmail、Yahooメールなど。
    学校のメールアドレスや学校のアカウント（例：〇〇高校のアカウント、oughアカウントなど）は使用できません。
    学校アカウントは、外部サービスとの連携が制限されており、 GitHub や Streamlit にログインできない場合があるためです。

2. GitHubでリポジトリを作る

2-1. リポジトリとは？

リポジトリ（repository）とは、GitHub上に作る「プロジェクト用のフォルダ」のようなものです。
1つのアプリや1つの作品ごとに、1つのリポジトリを作って、その中にファイルをまとめて保存します。
このページで作る app.py や requirements.txt も、すべて同じリポジトリの中に置きます。

2-2. リポジトリ作成手順

  1. GitHubにログインし、右上の「＋」→ New repository をクリック
  2. Repository name に例：my-streamlit-app を入力する
  3. Public を選択する（公開したい場合）
  4. 必要なら「Add a README file」にチェックを入れる
  5. Create repository をクリックする

3. アプリ用ファイルと拡張子の説明

3-1. 拡張子とは？

ファイル名の最後についている .py や .txt の部分を 拡張子（かくちょうし） と呼びます。
拡張子によって、「このファイルは何の種類か（プログラムなのか、文章なのか）」を区別します。

  • .py（ピー・ワイ）
    Pythonというプログラミング言語で書かれたプログラムファイルです。
    Streamlitアプリの本体は app.py という名前の .py ファイルとして作ります。
  • .txt（テキスト）
    普通の文字だけが書かれたテキストファイルです。
    Streamlitでは、アプリで使うライブラリの一覧を requirements.txt という .txt ファイルに書いておきます。

3-2. app.py のサンプルコード

import streamlit as st

st.title("勇者のステータス表示アプリ")

namae = st.text_input("勇者の名前を入力してください")
level = st.slider("レベル", 1, 99, 10)
hp = st.number_input("HP", min_value=1, max_value=999, value=100)

if st.button("ステータスを表示"):
    st.write("=== 勇者のステータス ===")
    st.write("名前：", namae)
    st.write("レベル：", level)
    st.write("HP：", hp)

3-3. requirements.txt の内容

streamlit

今回は streamlit だけを使うので、1行だけでOKです。

4. GitHub上でファイルを作成する（Create new file）

4-1. app.py を作成

  1. 作成したリポジトリのページを開く
  2. ファイル一覧の右上にある Add file → Create new file をクリック
  3. 「File name」に app.py と入力する
  4. 下の大きなテキストエリアに、先ほどの app.py のコードを貼り付ける
  5. 下にスクロールして「Commit new file」をクリックし、保存する

4-2. requirements.txt を作成

  1. もう一度 Add file → Create new file をクリック
  2. 「File name」に requirements.txt と入力する
  3. テキストエリアに次の1行を書き込む
streamlit
  1. 再び「Commit new file」をクリックして保存する

これで、リポジトリの中に app.py と requirements.txt がそろった状態になります。

5. Streamlit Community Cloudで公開する

  1. Streamlit Community Cloud にアクセスし、GitHubアカウントでログインする
  2. New app ボタンをクリックする
  3. Repository に、先ほど作ったリポジトリ（例：ユーザー名/my-streamlit-app）を選ぶ
  4. Branch は通常 main、Main file path は app.py を指定する
  5. Deploy を押すとビルドが始まり、完了すると https://〜.streamlit.app のようなURLが発行される

このURLを共有すれば、他の人もブラウザからアプリを使うことができます。

6. この説明ページをGitHub Pagesで公開する

  1. 同じリポジトリの中に、この index.html と style.css を作成する
  2. リポジトリの Settings → Pages を開く
  3. Source の Branch を main、フォルダを /root に設定して保存する
  4. 数十秒〜数分後、GitHub Pages のURL（例：https://ユーザー名.github.io/リポジトリ名/）が表示される

そのURLにアクセスすると、この説明ページがWebサイトとして表示されます。
